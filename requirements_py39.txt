# Glot500 requirements - cleaned for `pip install -r` from PyPI.
#
# Lines commented out below cannot be installed by plain `pip install` from PyPI
# (wrong name, not published on PyPI, needs a special index, or needs to be built
# from source). Install them separately after the rest succeeds - and only if
# your specific workflow actually needs them.
#
# Recommended Python: 3.9 (per the Glot500 README).
#
# Suggested install order:
#   1) python -m venv .venv && source .venv/bin/activate
#   2) pip install --upgrade pip
#   3) pip install torch==2.0.0           # (use the right --index-url for your CUDA)
#   4) pip install -r requirements_clean.txt
#   5) Handle the commented-out packages manually (see notes per line).

# --- Packages NOT installable from PyPI as written -------------------------

# apex==0.9.10dev
#   NVIDIA Apex is not published on PyPI. The string "0.9.10dev" is what
#   NVIDIA's setup.py reports after a source build. Install from source:
#       git clone https://github.com/NVIDIA/apex
#       cd apex
#       pip install -v --disable-pip-version-check --no-build-isolation \
#         --config-settings "--build-option=--cpp_ext" \
#         --config-settings "--build-option=--cuda_ext" ./
#   Only needed if you run the pretraining scripts that import apex.

# faiss==1.5.3
#   Meta publishes FAISS on PyPI as `faiss-cpu` or `faiss-gpu`, not `faiss`.
#   Replace with:  pip install faiss-cpu==1.7.4   (or faiss-gpu, or conda).

# torch_xla==1.0
#   Google TPU backend. Version 1.0 is incompatible with torch 2.0.0, and it
#   ships from a special wheel index for TPU hosts. Skip unless you train on TPU.

# mxnet==1.9.1
#   Apache MXNet is retired; wheels often don't exist for Python 3.9+.
#   Only install if a specific preprocessing script needs it; consider an
#   isolated Python 3.7/3.8 env for that step.

# gluonnlp==0.10.0
#   Depends on mxnet. Install only if you actually run a script that imports it.

# torchdynamo==1.13.0
#   TorchDynamo was merged into PyTorch starting with torch 2.0 (now
#   `torch._dynamo`). Installing the standalone package conflicts with torch 2.0.

# intel_extension_for_pytorch==2.0.100
#   Intel-specific accelerator. Needs Intel's wheel index and Intel hardware
#   to be useful. Skip on NVIDIA-GPU / generic CPU setups.
